# 기획 재정의 — 정보 전달기 → 관람 동행자 (Companion Reframe)

> 상태: **브레인스토밍 / 방향 설정** (구현 아님)
> 작성: 2026-07-07
> 맥락: 로그인 게이트·AI 추천 결과 표시까지 붙인 시점. 개발보다 기획·UX를 다시 본다.
> 관련: `onboarding-redesign-2026-06`(온보딩 재설계 제약), `onboarding-flow-qa.md`

---

## 1. 문제 인식

현재 Roam은 **"AI가 박람회 가이드"**를 표방하지만, 실제 사용자 경험은
**동행자(companion)가 아니라 정보 전달기(information delivery)**에 가깝다.

- 사용자가 묻기 전에 AI가 **완성된 동선을 던진다** → 사용자는 받는 사람일 뿐, 판단 주체가 아님.
- "왜 이 부스인지"의 근거보다 "여기 가라"는 결과가 앞선다.
- 관람이 끝나도 *"충분히 즐겼다 / 잘 봤다"*는 감각을 설계하지 않았다. 목표 달성만 있고 경험이 없다.

**핵심 전환**: 앱이 사용자를 *대신 판단*하는 게 아니라, 사용자가 *스스로 잘 판단하도록 기반을 다져주는* 존재가 되어야 한다.

---

## 2. 핵심 재정의 (From → To)

| | From (현재) | To (지향) |
|---|---|---|
| 앱의 역할 | 동선 생성기 | **판단 근거 제공자 = 동행자** |
| 주 산출물 | 최적 동선(route) | **부스 이해 + 결정 지원**. 동선은 부산물·도구로 강등 |
| 상호작용 | 온보딩 폼 → 결과 1회 | **관심 기반 피드**로 지속 상호작용 |
| 정보 | 답을 준다 ("여기 가라") | **기준을 준다** ("이 시간대 붐빔, 이유는 굿즈 이벤트") |
| 실시간 | (약함) | 현장에서 *즉각 판단할 기준*만. 명령 아님 |
| 지도 | 중심 인터페이스 | **부가 서비스** — 온사이트 공간 참조용. 정보·사이드바는 앞단에서 해결 |
| 시간축 | 1회성 관람 | **관람 이력 누적** → 다음 박람회 경험이 더 좋아짐 (리텐션·해자) |
| 목표 | 앱이 정함 | **사용자가 개인 목표를 설정·향상** |

---

## 3. 은유 — 개인 맞춤 PT

> 좋은 PT는 대신 운동해주지 않는다. 세팅을 잡고, 자세를 교정하고, 페이스를 맞추고, 끝나면 "오늘 잘했다"고 확인시킨다.

동행자 Roam도 같다:
- **세팅**: 전시의 큰 그림을 먼저 이해시킨다 (오리엔테이션).
- **자세 교정**: 사용자가 헤매지 않도록 기준을 준다. 대신 결정하진 않는다.
- **페이스**: 시간·혼잡에 맞춰 무리 없이.
- **마무리 확인**: 관람 후 "충분히 즐겼다"는 감각을 설계.

---

## 4. 설계 원칙

1. **판단 근거 > 판단 결과.** 정답을 주지 말고 결정할 재료를 준다.
2. **앞단에서 해결.** 정보·맥락·추천은 지도 진입 전에 소화시킨다. 지도는 순수 공간 참조.
3. **명령 아닌 큐(cue).** 실시간은 "피하라"가 아니라 "이래서 붐빔" — 판단은 사용자 몫.
4. **일관된 상호작용으로 공간 경험 증강.** 단발 목표 달성이 아니라 계속 이어지는 관계.
5. **끝의 감각을 설계.** 헤매지 않게 + 다 보고 났을 때 만족. 성취가 아니라 경험.
6. **누적이 곧 개인화.** 관심·이력이 쌓일수록 추천도, 사용자 자신의 안목도 자란다.

## 5. 핵심 표면 / 메커니즘

### a. 오리엔테이션 — "전시 큰 개념 먼저"
- 세부 동선보다 **전시의 개념·규모·홀/테마 구조**를 빠르게 이해시키는 걸 우선.
- 사용자에게 *멘탈 모델*(이 전시는 뭐고, 어떻게 나뉘고, 나는 어디에 관심)을 먼저 심는다.
- 세부는 피드에서 해결.

### b. 관심 피드 (쓰레드형)
- 부스 정보를 **사전 노출**. 관심 기반 추천 피드.
- **한 번에 6개 미만.** 관심 항목 클릭 → 관련 부스 추천 (쓰레드처럼 확장).
- 관심 피드가 **누적** → 사용자의 **학습도(안목) 지표**로 활용.
- 이 피드가 온보딩의 "의도 묻기"를 상당 부분 흡수한다 (§8 긴장 참조).

### c. 근거 카드 — 판단 기반
- 각 부스/추천에 "왜"를 붙인다: 무엇을 하는 곳인지, 왜 너와 맞는지, 무슨 근거인지.
- 사용자가 **스스로 갈지 말지 결정**할 수 있을 만큼의 재료.
- ⚠️ 이건 **부스 콘텐츠(enrichment)에 의존**. 현재 79/256(31%) 채워짐 — 빈 상태 아님. 갭 = 커버리지 + 근거카드용 필드 충분성 (§8).

### d. 실시간 판단 큐 (명령 아님)
- 현장에서 **즉각 판단할 기준 정도만** 제공. 예:
  - "이 시간대엔 많이 붐빈다."
  - "이 부스 ~ 굿즈 이벤트 때문에 붐빈다."
  - "00시부터 이 부스에서 ~ 이벤트 예정."
- "가지 마라"가 아니라 사실 + 이유. 결정은 사용자.

### e. 개인 목표 설정
- 사용자가 **자기 목표를 설정**하고, 서비스가 그 목표 달성을 돕고 향상시킨다.
- 앱이 정한 KPI가 아니라 사용자 자신의 관람 목표.

### f. 종단 프로필 — 이력 누적
- 박람회 경험 데이터를 **누적**. 계속 쓰면 앞으로의 박람회 경험도 더 좋아짐이 **예상되게** 유도.
- 관심 피드 누적 = 학습도. 이력 = 다음 추천의 연료 + 사용자 안목의 성장 기록.
- 이게 리텐션·해자의 근거.

### g. 지도 = 부가
- 전체 지도 인터페이스는 **그대로 유지**하되, 그 외 정보·사이드바는 **전부 앞단에서 해결**.
- 지도는 온사이트 공간 참조라는 **부가 서비스**로만.

## 5-B. 관람 아크 — "충분히 즐겼다" 감각 설계

> 전·중·후를 하나로 잇는 경험 설계. **만족은 끝화면이 아니라 3막의 페이오프.**
> 시작이 건 *약속*을 끝이 *갚는* 구조 — 끝에서 만족을 만들 순 없고, 시작에서 심은 기대를 회수한다.

### 근거 (인지 원리)
- **Peak-end rule**: 경험은 *절정 + 끝*으로 기억된다. 끝이 전체 만족을 과대결정 → 마무리 화면이 최고 레버리지.
- **Zeigarnik**: 미완은 계속 걸린다. "놓쳤다" 느낌이 남으면 불만족 → 종료는 "덜 봤다"가 아니라 "도착했다"로 닫아야.
- **Completeness = 프레임 의존**: 뭐가 있었는지 틀이 없으면 "충분함"도 없다 → §5-a 오리엔테이션과 직결.

### 이음새 = 두 가닥
1. **개인 목표/약속 실** — 시작에 심고, 중간에 조용히 추적, 끝에 회수.
2. **순간 누적** — 방문·메모·사진. 끝화면이 되짚을 재료.

### 3막

**1막 · 전 (약속 세팅) — 이중 트랙 (결정됨)**
- **온보딩에 명시적 개인 목표 스텝**: "오늘 이거면 만족" 계약을 사용자가 직접 설정.
- **피드가 암묵적으로 목표 유도**: 관심 클릭·누적이 목표를 은근히 형성·보정.
- 둘 다 필요 — 명시(선언) + 암묵(행동)이 서로 보정.
- ⚠️ **약속은 가용 시간에 맞게 작게.** 30분인데 20부스 약속 → 끝에 무조건 실패감.

**2막 · 중 (진행감 축적)**
- 방문 부스 = 이야기의 한 박자. 체크가 아니라 *뭘 얻었나*.
- 진행 신호는 **앰비언트**. 나그지 않는다(원칙3). 조용히 쌓인다.

**3막 · 후 (회고·의미·연결)**
- Peak-end 해소: 나선 것 → 실제 경험 → 의미로 되짚기.
- 놓친 것 = 실패가 아니라 **"다음"으로 리프레임**(Zeigarnik 해소).
- 종단 프로필로 핸드오프 → "다음 박람회가 더 좋아짐"(§5-f).

### 회고 재료 — 자발 + 자동 합성 (결정됨)
- **자발**: 사용자 메모·사진 (개인 메모장).
- **자동 캡처**: 부스 방문·체류시간·순간(도착·이벤트 조우 등).
- **둘을 합쳐 분석** → 서사 리캡 생성. 자발만 의존 시 빈약, 자동만 의존 시 무미 → 합쳐야 의미 있는 회고.

### 결정적 레버 — 분모 프레이밍
- "충분히 봤다 vs 덜 봤다"는 **분모가 가른다.**
- 나쁨: "전체 100부스 중 9개" → 91개 놓침 = 불만족.
- 좋음: "너한테 맞는 12곳 중 9곳" → 거의 다 = 충족.
- 끝화면 분모 = 전체가 아니라 **개인화된 관련 집합**. 프레임이 만족을 만든다.

### 끝 경험 방식 (결정됨)
- **채택: A(서사 리캡) + C(다음으로 잇기).** "오늘의 관람" 이야기 = 목표 대비 + 남긴 순간 → 종단 프로필 연결.
- **배제: B(성취 대시보드·뱃지·점수).** 게임화 함정(§8-4). "즐김"을 "달성"으로 후퇴 = 재정의 원칙5 위반.

### 트리거 재설계
- 현재 `map-view.tsx` 뒤로가기 → "관람 끝났나요?" → completed = **기능적 하차, 경험적 닫힘 아님.**
- "네가 그만뒀다"가 아니라 "**네가 도착했다**" 느낌으로.
- 오픈: 자연 종료 감지(목표 근접·시간 소진·출구 근처) 후 선제 제안 vs 순수 사용자 트리거 유지 — 미정.

## 6. 성숙 단계 — 고관여 사용자

- 서비스가 성장하면 **전문성 높은 = 고학습·고관여 사용자** 대응도 고려.
- 초보: 오리엔테이션·큰 개념 중심. 고관여: 깊은 정보·자기 큐레이션·목표 세분화.
- 종단 프로필(§5-f)이 초보 → 고관여로의 성장 경로가 된다.

## 8. 긴장 · 리스크 · 오픈 퀘스천

1. **enrichment 커버리지·필드 충분성.** "근거를 준다"는 비전은 부스 콘텐츠 의존. 현재 79/256(31%) — 빈 상태 아님. 남은 커버리지 + 근거카드/confidence용 필드가 지금 스키마로 충분한지가 실제 과제. 수동 입력 런북 `docs/booth-enrichment.md`.
2. **온보딩 3번째 피벗.** 폼 → 대화 → 피드. 방금 대화형 재설계 끝냈음(`onboarding-redesign-2026-06`). 피드가 온보딩을 **대체**하나 **공존**하나? 대화형 자산을 버릴지 재활용할지 결정.
3. **피드 <6 + 관심기반 = 필터버블.** 전시는 유한 공간 → 발견(serendipity) 슬롯을 일부러 섞어야. "관심 밖인데 놓치면 아까운" 것 노출 규칙 필요.
4. **학습도 지표의 용도.** 사용자에게 어떻게 보여줄지 미정. 게임화 함정(뱃지·점수 남발) 경계. 안목의 성장 = 내적 보상으로.
5. **동선 강등 vs 기존 UX.** 뒤로가기 "관람 끝났나요?" 등 동선 전제 플로우 다수. 부산물화하면 이 흐름들 재설계.
6. **~~"충분히 즐겼다" 감각의 구체화~~ → §5-B에서 설계 완료.** 남은 오픈: 종료 트리거 방식(선제 감지 vs 사용자), 자동 캡처 범위.
