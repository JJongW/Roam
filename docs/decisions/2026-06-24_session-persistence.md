# 세션/기기 유지 시 입력 데이터 보존

**날짜**: 2026-06-24
**관련 파일**: stores/route.ts, stores/onboarding.ts

## What — 무엇이 달라졌나
- 사용자가 입력·생성한 정보(담은 부스·메모·사진·생성된 동선·온보딩 답변)가, 같은 기기에서 앱을 닫았다 다시 열어도 그대로 남는다. 탭을 닫아도 유실되지 않는다.
- 초기화는 오직 "동선 비우기"로만 명시적으로 한다.

## Why — 왜 필요했나
- 동선·온보딩 답변이 탭 세션(sessionStorage)에만 있어, 탭을 닫으면 사라졌다. 사용자가 "세션/폰이 유지되면 입력한 모든 정보를 유지해 달라"고 요청.

## 판단 근거
- 담은 부스·메모는 이미 기기 단위(localStorage)로 남고 있었는데, 동선·온보딩만 세션 단위라 일관성이 없었다. 모두 기기 단위로 통일해 "기기를 유지하면 데이터도 유지"를 만족.
- 오래된 동선이 남는 문제는 자동 삭제가 아니라 사용자가 직접 누르는 "동선 비우기"로 다루기로 이미 정했고(이전 결정), 그 원칙을 유지한다 — 자동 유실보다 명시적 초기화가 신뢰 가능.
