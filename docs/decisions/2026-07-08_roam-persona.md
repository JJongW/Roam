# Roam 페르소나 정의서 (1p)

_2026-07-08 · companion-reframe Phase D. 문구 재스킨의 단일 근거._

## 정체성

**Roam** — 전시·박람회를 **같이 걷는 관람 동행자**. 정보 안내기(챗봇)가 아니라,
흩어진 부스 정보를 사용자의 관심·가치 언어로 번역해 "이게 왜 나한테 끌리지"를
**스스로 판단하게** 돕는 존재. 아바타 = 걷는 R 글리프 로고(`/logo.svg`).

## 목소리 (단일 규칙)

- **반말·친근**. 존댓말·반말 혼용 금지 — UI 텍스트 전부 반말로 통일한다.
- **1인칭 발화**. UI 문구는 시스템 설명이 아니라 Roam의 말. "추천 결과" 아니라
  "내가 미리 둘러보고 골라둔 곳". 라벨도 발화체.
- **판단은 사용자, Roam은 재료만**. 명령("가지 마") 금지. **사실 + 이유 + 판단 기준**만
  주고 결정은 사용자에게 넘긴다. (실시간 큐 원칙과 동일 — `cue.ts`.)
- **영업·과장 금지**. "최고!"·"놓치면 후회" 류 X. 담백하게 왜 맞을 수 있는지만.
- **짧게**. 한 호흡. 부제/설명은 한 줄.

## 하지 말 것

- "안정픽/낯선픽/모험픽" 같은 알고리즘 내부 용어를 사용자에게 그대로 노출 → 발화로 번역.
- "~하세요"·"~해주세요" 존댓말.
- 동선·최적화·효율 프레이밍(재정의로 폐기).

## 적용 범위 (Phase D)

문구 재스킨만. **로직 0 변경**. 대상: `value-onboarding`(존댓말 잔재 제거),
`interest-feed`(PickLabel 발화화), 홈 부제·온보딩 게이트(이미 반말). 반응 버튼
라벨(끌림/나중에/별로/이미 봄)은 사용자 행동이라 유지.
