# 방문 목적 다중 선택

**날짜**: 2026-06-22
**관련 파일**: wizard.tsx, onboarding(store), route-view.tsx, ai-quick-start.tsx, route-preferences.ts, scoring.ts, constants.ts, schemas, types, repository, 0011 마이그레이션

## What — 무엇이 달라졌나
온보딩의 "방문 목적이 무엇인가요?"에서 이제 목적을 여러 개 고를 수 있다(예: 구매 + 체험). 고른 목적들이 모두 추천 동선에 반영되고, AI 자유서술 추천도 글에서 여러 목적을 함께 읽어낸다.

## Why — 왜 필요했나
실제 방문객의 목적은 하나가 아닌 경우가 많다(사러 왔지만 체험도 하고 싶고, 정보도 얻고 싶다). 그런데 화면에서는 목적을 단 하나만 고를 수 있어, 자신을 제대로 표현하지 못하고 추천도 한쪽으로만 치우쳤다. 여러 목적을 가진 사람이 그중 하나를 포기하고 골라야 했다.

## 판단 근거
- 사람의 방문 동기는 복합적이다. 단일 선택은 현실을 단순화해 추천 품질을 떨어뜨린다.
- 여러 목적을 고르면 그 목적들을 고루 반영한 동선을 주는 편이, 하나만 강제하는 것보다 "내 의도대로 골라줬다"는 느낌을 준다.
- 관심사 선택이 이미 다중 선택이라, 목적도 같은 방식으로 고르게 해 화면 사용법의 일관성을 지켰다.
- AI 자유서술 추천도 동일하게 여러 목적을 인식하게 해, 두 입력 경로(직접 선택 / 자유서술)의 결과가 어긋나지 않도록 했다.
